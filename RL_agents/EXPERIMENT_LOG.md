### 4.3 交易次数为 0 的根因分析

**问：交易次数为 0 是代码 bug（engine_v2.py 持仓标志硬编码问题）还是学出来的策略？**

答：是算法学出来的策略，不是 bug。engine_v2.py 的持仓标志问题只影响实时推理引擎，
训练和评估走的是 env.py + ppo.py 的 evaluate() 函数，持仓状态是正确动态维护的。

根因分析（reward v2 的 absolute 模式）：
```
reward = (V(t) - V(t-1)) / V(t-1)
```
- 空仓 + 市场涨 → reward = 0（现金不变，无惩罚）
- 空仓 + 市场跌 → reward = 0（现金不变，无惩罚）
- 持仓 + 市场涨 → reward > 0
- 持仓 + 市场跌 → reward < 0
- 任何交易 → 扣手续费，立即产生负 reward

在信号噪声大（128 天稀疏数据）的条件下，agent 的理性选择：
> "交易有手续费（确定的负 reward），但收益不确定。不交易稳拿 0。"
→ 收敛到"躺平"的局部最优。

## 实验 5: Reward v3 — Relative Reward

### 5.1 设计动机
为解决 v2 中空仓无惩罚导致的"躺平"问题，引入相对基准收益：
```
reward = portfolio_return - benchmark_return
```
其中 benchmark_return = 当日股价涨跌幅（即 Buy & Hold 的单日收益率）。

设计意图：
- 持仓 + 市场涨 → reward ≈ 0（跟上了基准，中性）
- 空仓 + 市场涨 → reward < 0（踏空，被惩罚）← 关键改进
- 空仓 + 市场跌 → reward > 0（躲过下跌，被奖励）
- 持仓 + 市场跌 → reward ≈ 0（与基准同跌，中性）

### 5.2 消融实验 Round 2：Reward 模式对比 (贵州茅台)
| 配置 | 收益率 | 交易次数 | Sharpe |
|------|--------|---------|--------|

Relative reward 成功激活了 A2C 的交易行为（0→1 次），跑赢了 B&H。
但 PPO 因 clipped update 更保守，128 天数据仍不够触发交易。

### 5.4 Reward 与 Return 脱节问题

**问：为什么训练日志中 Return +326%、Sharpe 1.51 的 episode，reward 却是负值（-0.5746）？**

这不是 bug，是 relative reward 设计的副作用。Return 和 Reward 衡量的是不同的东西：
- **Return** = 组合的绝对收益率（+326% 意味着赚了钱）
- **Reward** = 每步 `portfolio_return - benchmark_return` 的累加（衡量每步有没有跑赢 B&H）

在整体上涨的训练区间里：
- 持仓的每一天：组合涨 ≈ 基准涨 → reward ≈ 0
- 空仓/交易的每一天：组合不动，基准在涨 → reward < 0
- 只有空仓 + 基准下跌时 reward > 0

因此累积 reward 在牛市里几乎不可能为正。agent 赚了 +326%，但 relative reward 说"你每一步都没跑赢基准"。
**这会让训练信号嘈杂，agent 难以区分"好策略"和"坏策略"。**


## 实验 6: Reward 函数文献调研

### 核心发现：我们的 reward 设计偏离了文献主流做法

#### Yang et al. (ICAIF 2020) / FinRL 框架的 reward
```python
# FinRL env_stocktrading.py 原始代码
end_total_asset = cash + sum(price_i * shares_i)
begin_total_asset = ...  # 上一步的组合价值
self.reward = end_total_asset - begin_total_asset
self.reward = self.reward * self.reward_scaling  # 默认 1e-4
```

关键区别：
1. **FinRL 用的是原始差值 `V(t) - V(t-1)`，不是收益率**
   - 我们的 v2: `(V(t) - V(t-1)) / V(t-1)` ← 做了归一化
   - FinRL: `V(t) - V(t-1)` ← 不做归一化
   - 归一化消除了"仓位规模"信号：持有更多资产时同样涨幅产生更大 reward，自然激励持仓
   - 归一化后这个激励消失了
2. **FinRL 不用 relative reward**
   - 不减去 benchmark_return
   - "不交易"问题不通过 reward 解决，而是靠 ensemble 选择器淘汰不交易的 agent
3. **FinRL 加了 reward_scaling（默认 1e-4）** 控制数值范围，避免梯度问题

#### Yang et al. 论文中的正式定义 (Equation 4)
```
r(s_t, a_t, s_{t+1}) = (b_{t+1} + p_{t+1}^T · h_{t+1}) - (b_t + p_t^T · h_t) - c_t
```
- b = 现金余额, p = 股价向量, h = 持仓向量, c = 交易成本
- 本质还是 delta portfolio value，交易成本自然体现在组合价值变化中

#### Differential Sharpe Ratio (Moody & Saffell, NeurIPS 1998)
```
D_t = (B_{t-1} · ΔA_t - 0.5 · A_{t-1} · ΔB_t) / (B_{t-1} - A_{t-1}²)^{3/2}
```
- A_t = 收益率的指数移动平均（一阶矩）
- B_t = 收益率平方的指数移动平均（二阶矩）
- η = 适应率（如 0.004）
- D_t = Sharpe ratio 对当前交易的边际贡献
- 优点：直接优化风险调整后收益，不交易 = 零收益 = 差 Sharpe = 负 reward
- 缺点：训练早期数据不足时噪声大，需要 warm-up

#### Deng et al. (2017) — 位置感知 reward
```
r(t) = a_t · log(p(t)/p(t-1)) - c · |a_t - a_{t-1}|
```
- a_t 是仓位 (-1, 0, +1)，第二项惩罚频繁换手

#### Risk-Aware Composite Reward (2025)
```
R = w1 · R_ann - w2 · σ_down + w3 · D_ret + w4 · T_ry
```
多目标加权：年化收益 - 下行风险 + 超额收益 + Treynor 比率。最全面但超参数多。

### 文献中 Reward 设计汇总
| 方法 | 公式 | 解决躺平？ | 复杂度 |
|------|------|:---:|:---:|
| FinRL / Yang et al. | `V(t) - V(t-1)` | 否（靠 ensemble 淘汰） | 低 |
| Differential Sharpe | `D_t`（边际 Sharpe 贡献） | 是（零收益 = 差 Sharpe） | 中 |
| 位置感知 (Deng) | `position · log_return - 换手惩罚` | 否 | 低 |
| 复合风险感知 | 多目标加权 | 部分 | 高 |

### 我们的三版 Reward 对比文献的问题
| 版本 | 公式 | 问题 | 文献对比 |
|------|------|------|---------|
| v1 | `log_return - 交易成本惩罚` | 躺平 | 类似 Deng，但缺少仓位项 |
| v2 | `(V(t)-V(t-1)) / V(t-1)` | 归一化消除仓位规模信号→躺平 | 接近 FinRL 但不该归一化 |
| v3 | `portfolio_return - benchmark_return` | 牛市 reward 永久为负，信号嘈杂 | 文献中无此做法 |

### 结论
建议的修改方向：
1. **Reward 回归 FinRL 主流**：`reward = (V(t) - V(t-1)) * reward_scaling`，不归一化，不减基准
2. **"不交易"问题交给 ensemble 层**：用 Sharpe ratio 作为选择器指标，淘汰不交易的 agent
3. **可选进阶**：Differential Sharpe Ratio 作为实验对照

## 实验 7: DDPG Agent 实现与三算法消融 (稀疏数据)

### 7.1 DDPG 实现
新建文件:
- `src/rl/replay_buffer.py` — off-policy 环形经验回放缓冲区
- `src/rl/ddpg.py` — DDPGActor/Critic + Target 网络 + OU 噪声 + 训练器

- 持仓观测从二值 (0/1) 改为连续比例 (stock_value / portfolio_value)

DDPG 与 PPO/A2C 的核心区别:
- 连续动作 [0,1] = 目标仓位比例（可表达"买入30%仓位"）
- Off-policy + Replay Buffer，样本效率更高
- Actor/Critic 分离网络 + Target 网络 polyak 软更新
- OU 噪声探索 + 噪声衰减

单元测试: 新增 10 个测试（连续环境3 + ReplayBuffer2 + 网络2 + 训练器3），共 28 个全部通过。

## 实验 9: Reward v4 (FinRL 风格) + Ensemble + Rule-Based 风控

### 9.1 Reward 函数改为 FinRL 风格

改动 `env.py`，新增 `reward_mode="delta"`（设为默认）:
```python
reward = (V(t) - V(t-1)) * reward_scaling  # scaling 默认 1e-4
```

与之前的区别:
| 版本 | 公式 | 问题 |
|------|------|------|
| v2 absolute | `(V(t)-V(t-1)) / V(t-1)` | 归一化消除仓位规模信号 → 躺平 |
| v3 relative | `portfolio_return - benchmark_return` | 牛市 reward 永久为负 |
| **v4 delta** | **`(V(t)-V(t-1)) * 1e-4`** | **不归一化，持仓越多涨幅越大 reward 越大** |

### 9.2 Rule-Based 风控 Agent

新建 `src/rl/risk_manager.py`:
- **回撤止损**: drawdown >= 10% → 强制清仓 + 5 天冷却期，到期自动解锁，peak 重置
- **波动率过滤**: 用训练集 80 分位数动态计算阈值，波动率过高时禁止开新仓（不强制卖出）
- 风控层在 ensemble 选出 agent 后叠加，不改变训练逻辑

### 9.3 Ensemble 选择器

新建 `src/rl/ensemble.py`，实现 Yang et al. (ICAIF 2020) 的滚动选优:
```
|<--- 420天训练 --->|<- 60天验证 ->|<- 60天测试 ->|
                     ↑ 三个agent各跑一遍，选Sharpe最高的
                                    ↑ 用选中的agent交易
```

Delta reward 是本轮最关键的改进:
1. **reward 信号强度大幅增强**: 数值从 ±0.01 跃升到 ±10~20，梯度信号清晰
2. **仓位激励恢复**: 持仓越多、涨幅越大 → reward 绝对值越大 → agent 学会重仓
3. **选择器准确率从 42% 跳到 62%**: 因为 agent 行为差异化更明显，验证期 Sharpe 的区分度提高

### 9.6 项目总结 — 从 0 到 Ensemble 的完整路径

架构层:
```
数据 → 特征工程(10因子) → TradingEnv(离散/连续) → RL Agents(PPO/A2C/DDPG)
    → Ensemble 选择器(验证期 Sharpe) → Rule-Based 风控(回撤止损+波动率)
```

关键里程碑:
1. 单 Agent PPO 基线 → 验证 RL 可行性
2. A2C 加入 → 验证不同算法擅长不同行情
3. DDPG 加入 → 连续仓位控制，交易活跃度大幅提升
4. 完整日线数据 → 数据瓶颈突破，agent 开始真正交易
5. Delta Reward → 仓位激励恢复，ensemble 效果质变
6. Ensemble + 风控 → 54% 胜率，超越所有单 agent

未解决的问题:
- 极端行情窗口 (B&H +427%) 仍然拖累平均 alpha
- 风控层 10% 止损阈值偏紧，需要调优
- 仅验证了单只股票 (302132.SZ)，泛化性待验证
- Reward 尚未切换到 FinRL 的非归一化原始差值（当前 delta 仍用 scaling）
